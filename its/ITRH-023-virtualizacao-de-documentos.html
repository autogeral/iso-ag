1. Objetivo

O projeto tem como objetivo virtualizar todos os nossos documentos, assim permitindo o acesso remoto, facilitando a busca pelos documentos, evitando perda de documentos e gastos de espaço e recursos.

4. Procedimentos e Responsabilidades

Será possível armazenar, organizar, acessar e encontrar documentos de forma segura e remota utilizando o OpenKM (Software que será utilizado para armazenar e gerenciar os documentos), nele, cada usuário terá sua devida função e permissão garantindo que não corra o risco de um documento ser acessado por um setor/usuário que não seja responsável por aquele documento.

4.1. Entrada.

4.1.1 Login no OpenKM

Assim que conectar no OpenKM, irá aparecer uma tela de acesso, pedindo por um usuário e senha (neste caso usaremos no login "OkmAdmin" e admin como senha)


4.2 Utilizando o Software

4.2.1 Primeiras Ações

Após o acesso, terá esta tela como retorno:


Nesta tela estão as principais funcionalidades, como:
-Gerenciamento de arquivos e pastas (criação, edição e exclusão)
-Visualização geral da aplicação
-Navegação entre os repositórios

4.2.2 Permissões

Antes de prosseguir com as instruções, uma breve explicação sobre as permissões, tudo o que o usuário poderá fazer dentro do software gira em torno da permissão concedida pelo administrador, ou seja, o usuário poderá ter permissões desde apenas leitura até criação de metadata e manipular arquivos. Portanto tudo irá depender da função na qual o administrador designou cada usuário

4.2.3 Instruções básicas

As primeiras possíveis ações dentro do software é justamente a criação de pastas e arquivos, para isso acesse a barra de menu


neste menu as ações mais básicas podem ser executadas, no caso:
-criar arquivos/pastas
-procurar por arquivos ou pastas
para isso basta clickar em "file"

Breve explicação dos Ícones:
-Find Folder = Procurar pasta
-Find Document = Procurar documento
-Create Folder = Criar pasta
-Add Document = Adicionar documento
-Write mail = enviar uma mensagem para outro usuário dentro da plataforma
-Create from template = Cria uma estrutura "de árvore" para organizar melhor arquivos de texti (TXT, HTML e PDF)
-Start Workflow = Começará um workflow (é uma sequência de processos, que passa do início ao fim)
-Refresh = Atualiza a aplicação rapidamente (semelhante ao f5 no navegador)
-Purge trash = Limpa toda a lixeira (exclui permanentemente os arquivos que lá estavam)
-Exit = Sai do software

4.2.4 Criação

Para criar uma pasta para guardar algum arquivo, acesse File → Create Folder

basta dar o nome desejado para a pasta e apertar a tecla enter, fazendo isto uma nova pasta será criada


Agora para Adicionar um documento, acesse file → Add Document → Escolher arquivo → Selecionar o Arquivo desejado

Pode também importar arquivos do tipo ZIP e também permite notificar outros usuários desta ação
Vale lembrar que pastas podem ser criadas dentro de outras pastas cada uma tendo seus arquivos, organizados de acordo com a preferência do usuário

4.2.5 Navegando pelo menu lateral

Agora será apresentado o menu lateral a esquerda da aplicação, esse menu ocupa uma boa parte do visual da aplicação

Vale destacar que nem todas as sessões desta aba podem ser gerenciadas pelo usuário comum. Agora uma breve explicação sobre cada sessão:
-Taxonomy: é a visualização geral dos repositórios, na qual poderá ser acessado e navegar entre as pastas e arquivos. Importante destacar que ele também funcíona na estrutura de árvore, ou seja, tem uma estrutura feita em remificações, por exemplo: Dentro da pasta PRINCIPAL, tem a SECUNDARIA e dentro dela a GERAL, e assim por diante
-Categories: exemplo de Sessão não gerenciável no máximo criar uma pasta e escrever um recado (mail)
-Metadata: outro caso de sessão sem permissão, no caso do usuário comum ele pode apenas visualizar as criadas e deixar um recado
-Thesaurus: mesmo caso da metadata
-Neste o usuário pode de fato criar uma pasta e adicionar arquivos para deixar esse template pronto para ser reutilizado
-My documents: neste o usuário também pode criar pastas e adicionar documentos, mas essa sessão tem como objetivo guardar documentos voltados ao usuário
-E-mail: funciona como qualquer outro email da web, porém este serve apenas para comunicação entre os usuários dentro da plataforma
-Trash: é a lixeira, onde ficará todos os arquivos e pastas deletados
