4. Procedimentos e Responsabilidades

Receba o FRH-006 preenchido, verifique se é solicitado a criação de uma conta Microsoft 365. Caso solicitado, siga toda a instrução, caso contrário, siga diretamente a partir do índice {[substituir pela referencia correta]}

4.1 Adicionar um usuário ao Microsoft 365

Pré-condição: receber o FRH-006 com o campo "Microsoft 365 (e-mail)" marcado em "SIM".
Acessar admin center do Microsoft 365
Acesse Usuários ativos - admin center do Microsoft 365 com sua conta de administrador;
Ao acessar, clique em "Adicionar um usuário".

Informações básicas
Conforme a imagem abaixo, em Adicionar um usuário, preencha os campos 'Nome' e 'Sobrenome' com as informações da FRH-006;
O campo 'Nome de exibição' é preenchido automaticamente ao terminar de escerever os campos anteriores;
Para o 'Nome de usuário' use o primeiro nome e último sobrenome separados por um ponto (.). Somente caso o nome de usuário não estiver disponível, crie um diferente, mas ainda usando o formato "nome.exemplo";
Você pode desselecionar a opção "Criar uma senha automaticamente" e para criar uma mais fácil de ser comunicada;
MANTENHA SELECIONADA A OPÇÃO "Exigir que este usário altere a senha quando entrar pela primeira vez";
Para evitar que seja necessário redefinir a senha em caso de esquecimento, destine para si mesmo a senha provisória deste usuário;
Clique em 'Avançar'.

Licenças do produto
Para que o usuário possa acessar os recursos com a conta Microsoft 365 é necessário que seja atribuída uma licença como a Microsoft 365 Business Basic, que dá acesso a aplicações online, acessíveis pelo navegador, incluindo Outlook, Excel, Word, Teams. Já a Microsoft 365 Business Standard disponibiliza algumas aplicações em suas versões desktop. Selecione a que mais se adapta as necessidades do colaborador, é preferivel que a versão Standard seja atribuída apenas a cargos de gestão. Mais abaixo na mesma tela, é possível verificar a lista de aplicativos disponibilizados com a licença selecionada.
Clique em 'Avançar'.

Configurações opcionais
Não fazemos uso das Configurações opcionais. Não é necessário preencher os campos de informações de perfil.
Clique em 'Avançar'.

Concluir
Revise e certifique-se de que as informações inseridas e selecionadas estão corretas.
Clique em 'Terminar de adicionar'.

4.2 Acesso ao sistema:

•Pré-Condição: receber o formulário de cadastro/mudança de funcionário para alteração no sistema:

-Acessar o banco de dados e definir o novo usuário através do comando: create user ‘< política de criação de usuário>’@’%’ identified by ‘< política de criação de senha> ’

• Política de criação de usuário: O login do usuário será definido de acordo com esta política, que determina que o login do usuário deverá ser < primeiro nome>.< último nome>, tudo com letras minúsculas.

Exemplo: Arthur Camocardo Corrêa, o login será arthur.correa.

• Política de criação de senha: A senha do usuário será definida de acordo com esta política, que determina que a senha do usuário deverá ser o primeiro nome do usuário, iniciando com letra maiúscula, concatenado ao carácter @ e ao ano vigente.:

Exemplo: Arthur Camocardo Corrêa criado no ano de 2019, a senha será Arthur@2019, recomenda-se a troca da senha no primeiro acesso, pelo usuário.

-Acessar o banco de dados e verificar as permissões do usuário que têm a mesma ou parecida função do novo funcionário através do comando: show grants for ‘< nome do usuário>’@’%’.

-Estas mesmas permissões deverão ser passadas ao novo usuário, para isso basta copiá-las e executá-las, não esqueça de mudar o nome do usuário a receber os novos privilégios.

-O próximo passo é a criação ou substituição do usuário no sistema ERPJ, abra a tela de cadastro de acessos, preencha os campos e clique no botão cadastrar, como ilustrado logo abaixo:

-Consulte no controle de acessos ao sistema, os níveis de acesso do usuário utilizado como base para criação do novo usuário.

-Se for substituição, consulte o usuário pelo padrão de login e altere os níveis de acesso conforme desejado, feito isso clique sobre o botão alterar.

-Caso o substituído estiver saindo da empresa, remova todos os acessos do mesmo

Cadastro de estoquista

Preencha com os dados do colaborador, e caso o mesmo utilize o coletor, deve-se informar no campo Coletor o número de série do equipamento.

4.3 Criação de e-mail, Gmail, LocaWeb

•Pré-Condição: receber o formulário de cadastro/mudança de funcionário no sistema e ter acesso de administrador aos e-mails.

•Gmail:

•Preencha os campos do formulário, a senha do usuário será definida de acordo com a política, que determina que a senha do usuário deverá ser o primeiro nome do usuário, iniciando com letra maiúscula, concatenado ao carácter @ e ao ano vigente.
Exemplo: Arthur Camocardo Corrêa criado no ano de 2019, a senha será Arthur@2019, recomenda-se a troca da senha no primeiro acesso, pelo usuário. e deve-se indicar a mudança de senha no primeiro acesso do usuário ao e-mail. Feito isso clique em adicionar novo usuário

•LocaWeb:

-Clicar em Criar caixa única
-Preencher os campos do formulário, seguindo as políticas de criação de usuários e senhas citados acima;

• Após criação do e-mail, o funcionário deverá ser incluído nos grupos de e-mails pertinente à sua função.
